# Decisions

本文件记录跨会话仍需保持一致的重要决策。新决策按日期追加，不重写历史。

## 2026-06-13：根目录直接作为教程仓库

- 决策：`/Users/air/Documents/codex开源仓库学习` 就是教程仓库根目录。
- 原因：避免再套一层目录，方便 Codex 桌面版项目和多会话共享上下文。
- 影响：所有课程章节、文档和测试都直接位于根目录。

## 2026-06-13：参考仓库不提交

- 决策：`learn-claude-code/` 与 `references/codex/` 仅作为本地研究材料，并写入 `.gitignore`。
- 原因：避免嵌套 Git 仓库、许可证内容和大量上游源码进入教程历史。
- 影响：事实映射和快照 SHA 必须写入 `docs/SourceMap.md`，不能只存在本地参考目录。

## 2026-06-13：Python 主线，Java 按价值引入

- 决策：章节默认只实现 Python 3.11+ 版本。
- 原因：目标读者需要先理解 Agent 机制，同时避免双语言实现拖慢主线。
- Java 引入条件：它必须显著帮助解释强类型事件、协议客户端或企业集成。

## 2026-06-13：不是 Rust 源码导读

- 决策：真实 Codex 源码用于校准设计，不作为章节叙事主线。
- 原因：目标读者目前不适合直接进入大型 Rust 异步系统。
- 章节顺序：问题 → 心智模型 → Python 教学实现 → 真实 Codex 对应 → 简化边界。

## 2026-06-13：每章一图、一机制、一提交

- 决策：每章开头必须有总结图；每章聚焦一个主要机制；完成后独立 commit 并 push。
- 原因：保持可学习性、可审阅性和多会话接力的清晰边界。

## 2026-06-13：每章必须先阅读对应 Codex 源码

- 决策：禁止凭印象或通用 Agent 知识直接编写章节。每章写作前必须阅读对应的 Codex 公开源码、测试与模块文档。
- 证据：每个完成章节必须包含 `SOURCE_NOTES.md`，记录源码快照、实际阅读文件、确认事实、教学简化与未确认项。
- 原因：教程的价值来自“基于 Codex 设计进行教学重建”，而不是泛化的 Agent 教程。
- 影响：没有 `SOURCE_NOTES.md` 的章节不得标记完成，结构检查也应报错。
